Halstead Software Metrics

Halstead's Theory (1971): Every program is made of operators and operands. mu1 = distinct operators (if, =, +, &&, function, etc.) mu2 = distinct operands (variables, constants, strings) N = N1 + N2 (total length) V = N x log2(mu1+mu2) (volume - mental effort to write it) B = E^(2/3) / 3000 (estimated bugs remaining at delivery) Note: Halstead was designed for assembly language (1971). Used here as a theoretical exercise - applied to JavaScript for academic purposes.

Function Points (Chapter 5)

Function Points measure HOW MUCH the Maternity Triage app does for users - language-independent. We count: inputs from user (EI), outputs with calculations (EO), simple data retrievals (EQ), and data stored (ILF). Since this app has NO external database, EIF = 0. Key difference: EO involves calculations. EQ is simple retrieval only.
